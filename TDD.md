# SleepLate 测试文档 (TDD)

## 1. 测试策略

### 1.1 测试原则
- **分层测试**: UI层、业务层、数据层独立测试
- **可维护性**: 测试用例清晰，失败信息明确
- **覆盖全面**: 核心功能高覆盖

### 1.2 测试类型
1. **单元测试**: 各模块独立功能
2. **集成测试**: 多模块协作
3. **UI测试**: 用户交互流程

## 2. 模块测试用例

### 2.1 ConfigService 配置服务

#### TC_CONFIG_001: 加载默认配置
```
输入: 配置文件不存在
预期: 返回默认配置 DefaultLockMinutes=30, Password=""
验证: 配置值正确
```

#### TC_CONFIG_002: 保存配置
```
输入: 有效AppConfig对象
预期: 配置文件成功写入
验证: 文件存在且内容正确
```

#### TC_CONFIG_003: 加载已保存配置
```
输入: 已存在的配置文件
预期: 正确读取各配置项
验证: 配置值与保存时一致
```

#### TC_CONFIG_004: 配置文件损坏处理
```
输入: 损坏的JSON配置文件
预期: 使用默认值，回退到初始状态
验证: 不抛异常，返回默认值
```

### 2.2 TimerService 定时器服务

#### TC_TIMER_001: 启动定时器
```
输入: minutes = 30
预期: 定时器启动，事件正常触发
验证:
- Start返回true
- Elapsed事件在指定时间后触发
- 剩余时间正确更新
```

#### TC_TIMER_002: 停止定时器
```
输入: 定时器运行中调用Stop()
预期: 定时器停止，不再触发事件
验证: Stop返回true，OnFinished不再触发
```

#### TC_TIMER_003: 重新设置定时器
```
输入: 定时器运行中设置新时长
预期: 重置倒计时，重新开始
验证: 从新时长开始倒计时
```

#### TC_TIMER_004: 超时后触发锁屏
```
输入: 倒计时结束
预期: OnFinished事件触发
验证: Lock()方法被调用一次
```

### 2.3 PasswordService 密码服务

#### TC_PASSWORD_001: 设置密码
```
输入: password = "123456"
预期: 密码加密存储
验证: 返回true，EncryptedPassword非空
```

#### TC_PASSWORD_002: 验证正确密码
```
输入: 已设置密码"123456"，验证"123456"
预期: 验证通过
验证: 返回true
```

#### TC_PASSWORD_003: 验证错误密码

```
输入: 已设置密码"123456"，验证"000000"
预期: 验证失败
验证: 返回false
```

#### TC_PASSWORD_004: 未设置密码时验证
```
输入: 未设置密码，验证任意值
预期: 返回false（首次运行需先设置）
验证: 返回false，日志记录警告
```

#### TC_PASSWORD_005: 空密码验证
```
输入: 密码为空字符串
预期: 验证失败
验证: 返回false
```

### 2.4 LockScreenService 锁屏服务

#### TC_LOCK_001: 执行锁屏
```
输入: 调用Lock()
预期: 系统锁屏
验证:
- 调用Win32API.LockWorkStation()
- 日志记录锁屏操作
```

#### TC_LOCK_002: 锁屏异常处理
```
输入: LockWorkStation()抛出异常
预期: 捕获异常，记录错误日志，程序继续运行
验证: 不崩溃，异常被正确处理
```

### 2.5 EncryptUtil 加密工具

#### TC_ENCRYPT_001: 加密字符串
```
输入: "test123"
预期: 返回非空加密字符串
验证: 加密后长度 > 原始长度
```

#### TC_ENCRYPT_002: 解密字符串
```
输入: 加密后的字符串
预期: 返回原始明文
验证: 解密结果与原始一致
```

#### TC_ENCRYPT_003: 加密解密一致性
```
输入: 任意字符串
预期: encrypt(decrypt(x)) = x
验证: 往返加解密保持不变
```

### 2.6 Logger 日志工具

#### TC_LOGGER_001: 记录不同级别日志

```
输入: DEBUG/INFO/WARN/ERROR级别消息
预期: 正确格式写入日志文件
验证:

- 格式正确: [时间戳] [级别] [模块] 消息
```

#### TC_LOGGER_002: 日志文件滚动
```
输入: 跨日期运行

```

#### TC_LOGGER_003: 多线程安全
```
输入: 多个线程同时写日志
预期: 日志不丢失、不损坏
验证: 文件内容完整，无交错
```

## 4. 集成测试用例

### 4.1 完整定时锁屏流程

#### TC_INTEGRATION_001: 定时锁屏完整流程
```
步骤:
1. 启动应用，主窗体显示
2. 设置锁屏时间为5分钟
3. 点击"开始定时"
4. 等待倒计时结束

预期:
- 定时器正常启动
- 倒计时正常更新
- 倒计时结束触发系统锁屏
- 日志完整记录

验证点:
- TimerService.OnTick事件正常
- 剩余时间正确递减
- LockScreenService.Lock()被调用
- 系统成功锁屏
```

#### TC_INTEGRATION_002: 取消定时流程
```
步骤:
1. 启动应用，设置5分钟
2. 点击"开始定时"
3. 等待1分钟后点击"取消定时"

预期:
- 定时器正常停止
- 锁屏不触发
- 日志记录取消操作
```

#### TC_INTEGRATION_003: 设置密码后退出
```
步骤:
1. 首次运行，设置密码为"abc123"
2. 再次点击退出
3. 输入正确密码

预期:
- 密码成功设置
- 验证成功
- 程序正常退出
```

## 5. 边界测试

### 5.1 输入边界

#### TC_BOUNDARY_001: 时间最小值
```
输入: 设置锁屏时间为0分钟
预期: 提示"时间不能为0"，使用默认值
验证: 不接受0值
```

#### TC_BOUNDARY_002: 时间最大值
```
输入: 设置锁屏时间为1440分钟（24小时）
预期: 允许设置
验证: 定时器正常启动
```

#### TC_BOUNDARY_003: 时间超出最大值
```
输入: 设置锁屏时间为2000分钟
预期: 提示超出范围
验证: 不接受超出值
```

#### TC_BOUNDARY_004: 非数字输入
```
输入: 输入"abc"
预期: 提示输入无效
验证: 不接受非数字
```

### 5.2 异常边界

#### TC_BOUNDARY_005: 配置文件只读
```
输入: 配置文件设置为只读
预期: 捕获异常，提示用户
验证: 不崩溃，日志记录
```

#### TC_BOUNDARY_006: 磁盘空间不足
